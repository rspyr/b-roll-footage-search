Key Tools & Frameworks for Your B-Roll Search App
Based on your requirements, here's a structured breakdown of the tech stack:

Core Architecture Components
1. Video Embeddings & Semantic Search
OpenAI CLIP - Vision-language model for encoding video frames with text queries. Fast and widely used for semantic video search
Google's VideoPoet or Meta's Timesformer - For temporal understanding of video sequences
Alternative: LLaVA or smolVLM - Lightweight vision-language models for local inference
2. Vector Databases
Milvus - Open-source, highly performant for scale
Weaviate - Built-in multimodal capabilities
ChromaDB - Simple, lightweight, good for prototyping
Qdrant - Fast and well-suited for production
3. Google Drive Integration & Sync
Google Drive API (Python client) - Official SDK for file operations
Auto-Sync-Drive - Ready-made periodic sync script with task scheduling
APScheduler or Celery - For scheduling the periodic sync checks
4. Video Processing & Transcription
FFmpeg - Frame extraction, format conversion
OpenAI Whisper - Audio transcription with high accuracy; runs locally or via API
AssemblyAI API - Cloud-based transcription alternative
MoviePy - Python wrapper for FFmpeg, easier integration
5. Backend & API
FastAPI - Modern Python framework, async support for handling jobs
Pydantic - Data validation
Python-dotenv - Environment management for API keys
6. Task Queue & Job Management
Celery - Distributed task queue for async video processing
Redis - Message broker for Celery; also useful for caching results
Alternative: RQ (Redis Queue) - Simpler than Celery if you don't need advanced features
